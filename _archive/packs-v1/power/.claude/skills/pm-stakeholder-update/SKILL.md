# PM Stakeholder Update Skill

## When to Use

Use this skill when:
- Writing weekly/monthly status updates
- Preparing executive summaries
- Communicating project progress
- Reporting blockers and risks
- Requesting decisions or approvals

## Examples

**Example 1: Weekly status update**
```
User: "Create a stakeholder update for this week"
Claude: [Uses pm-stakeholder-update skill] Creating update:

**Project Status: On Track** 🟢

**Completed This Week:**
- User authentication feature (100% complete)
- API integration testing (in progress, 80% complete)

**Next Week:**
- Complete API testing
- Begin dashboard UI implementation
- Security review scheduled

**Blockers:**
- Waiting on design assets for dashboard (ETA: 2 days)

**Metrics:**
- Velocity: 23 story points (target: 25)
- Burndown: On track for sprint goal
```

**Example 2: Executive summary**
```
User: "Create an executive summary of project status"
Claude: [Uses pm-stakeholder-update skill] Executive summary:

**TL;DR:** Project is 60% complete, on track for Q2 launch. One blocker identified requiring decision.

**Key Highlights:**
- Core features: 60% complete
- Timeline: On schedule
- Budget: 5% under budget

**Decision Needed:**
- Feature X scope reduction to meet deadline?
```

## Update Structure

This skill creates updates with:
- Clear status indicator (On Track, At Risk, Blocked)
- Completed work summary
- Upcoming milestones
- Blockers and risks (with mitigation plans)
- Key metrics and KPIs
- Action items and decisions needed

To use this skill:
- "Create a stakeholder update"
