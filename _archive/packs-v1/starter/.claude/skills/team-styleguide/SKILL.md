# Team Styleguide Skill

## When to Use

Use this skill when:
- Reviewing code for style consistency
- Making edits that should follow team conventions
- Ensuring code matches project styleguides
- Onboarding new team members to project standards

## Examples

**Example 1: Enforcing naming conventions**
```
User: "Update the function to use camelCase for variables"
Claude: [Uses team-styleguide skill] I'll update the function to follow camelCase naming conventions per your team styleguide.
```

**Example 2: Code formatting**
```
User: "Format this file according to our styleguide"
Claude: [Uses team-styleguide skill] I'll format the file using your team's styleguide rules (2-space indentation, trailing commas, etc.).
```

## Style Guidelines

This skill helps enforce:
- Naming conventions (camelCase, PascalCase, snake_case)
- Indentation and spacing rules
- Import organization
- Comment style and documentation standards
- File structure conventions

## Copy/Paste Ready

To use this skill, reference it in your prompts:
- "Use team-styleguide when reviewing this code"
- "Apply team-styleguide rules to this file"
- "Check if this matches our styleguide"
