# PM PRD Skill

## When to Use

Use this skill when:
- Writing new product requirements
- Updating existing PRDs
- Reviewing feature specifications
- Creating product documentation
- Preparing stakeholder communications

## Examples

**Example 1: Creating a new PRD**
```
User: "Create a PRD for user authentication feature"
Claude: [Uses pm-prd skill] I'll create a comprehensive PRD following standard structure:
- Executive Summary
- Problem Statement
- User Stories
- Requirements (functional & non-functional)
- Success Metrics
- Timeline & Dependencies
```

**Example 2: PRD review**
```
User: "Review this PRD for completeness"
Claude: [Uses pm-prd skill] Reviewing PRD against best practices:
✓ Problem statement is clear
✓ User stories follow INVEST criteria
✓ Success metrics are measurable
⚠ Missing technical constraints section
⚠ Timeline doesn't account for QA cycles
```

## PRD Structure

This skill ensures PRDs include:
- Clear problem statement and goals
- User personas and use cases
- Functional and non-functional requirements
- Success metrics and KPIs
- Technical constraints and dependencies
- Timeline and milestones
- Risk assessment

## Copy/Paste Ready

To use this skill:
- "Create a PRD for [feature]"
- "Review this PRD using pm-prd"
- "Update the PRD with new requirements"
