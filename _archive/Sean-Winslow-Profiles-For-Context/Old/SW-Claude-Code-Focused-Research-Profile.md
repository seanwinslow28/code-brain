## Research Requirements

### Must Include
1. **Sources**: Links, documentation, version numbers, dates
2. **How-To Steps**: Not just "what" but "how to implement"
3. **Examples**: Code samples, config files, templates
4. **Technical Explanations**: Why this approach? What are the tradeoffs? (I'm learning!)
5. **Context**: When to use each feature/approach

### What Makes Results Great
- Actionable immediately (can try it today)
- Teaches me something unexpected
- Shows me the "why" behind technical decisions
- Includes edge cases or advanced techniques
- Provides templates/blueprints I can adapt
- Challenges my assumptions

### Avoid
- Generic "getting started" guides (unless they reveal hidden depth)
- Outdated information without date context
- Obvious features without showing creative applications
- Abstract concepts without concrete implementation
- Assuming I know advanced concepts (explain as you go!)
