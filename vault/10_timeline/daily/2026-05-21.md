- **Pi platform overview (Topic 13 RE-RUN)** — Pi (pi.dev) is a minimal terminal coding harness created by Mario Zechner, now under Earendil Works (PBC). Open-source MIT, JS/TS extension system, hot-reloads agent-written extensions. Distinct from Raspberry Pi and Pi Network. Went viral in early 2026 after Armin Ronacher endorsed it. Differentiator vs Claude Code / Cursor: bring-your-own-skill philosophy + 4-tool minimalist harness (read/write/edit/bash) → [[20_projects/research/2026-05-21-topic-13-pi-platform-overview-gemini-dr-manual]]
- **Pi extensions catalog (Topic 14 RE-RUN)** — Official catalog at pi.dev/packages plus npm + Homebrew distribution. Notable for Sean's stack: `pi-ollama` (native Ollama provider, fixes streaming tool_call bug), `pi-agenticoding` (advisory context-compaction primitives), `@vtstech/pi-openrouter-sync` (OpenRouter model sync — flagged risky for cost-capped setups). Ecosystem is small but growing fast → [[20_projects/research/2026-05-21-topic-14-pi-extensions-catalog-gemini-dr-manual]]
- **Pi + OpenRouter (Topic 15)** — Pi has first-class OpenRouter support via `OPENROUTER_API_KEY` + `openai-completions` API type. Auth via `~/.pi/agent/auth.json` (0600 perms) OR shell-command resolution (`!security find-generic-password ...`) for macOS Keychain integration. Sean's 14-agent setup should use the Keychain pattern — never literal-string auth → [[20_projects/research/2026-05-21-topic-15-pi-openrouter-integration-gemini-dr-manual]]
- **Pi + local Ollama (Topic 16)** — `~/.pi/agent/models.json` with `baseUrl: http://mac-mini.local:11434/v1` + `api: openai-completions`. Set `OLLAMA_HOST=0.0.0.0` on Mac Mini for LAN access, Tailscale MagicDNS for off-LAN. Critical: set `num_ctx` in Modelfile (Ollama defaults to 2048; will silently truncate). DeepSeek-Coder-V3 and Qwen3-14B both tested for tool-calling reliability → [[20_projects/research/2026-05-21-topic-16-pi-ollama-integration-chatgpt-manual]]
