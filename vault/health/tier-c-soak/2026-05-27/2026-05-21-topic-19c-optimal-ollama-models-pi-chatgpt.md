This research report identifies the optimal Ollama models for Pi-driven coding and agentic workflows in 2026, categorized by hardware tiers based on available RAM and VRAM. The central thesis is that no single model dominates all aspects of the Pi workload—which includes tool-calling, code generation, multi-turn reasoning, and long-context summarization—necessitating a hardware-specific selection strategy to balance throughput, reasoning capability, and memory constraints.

The report provides specific model recommendations across three hardware tiers. For Tier A (M4 Max MBP, 48GB), **Gemma4:31b** is the best overall for agentic reasoning and tool-calling, while **Qwen3.5:35b-A3B-Coding** is a strong runner-up for high-speed coding and summarization, and **Qwen2.5-Coder:32b** is recommended when code accuracy is the primary priority. Tier B (M4 Pro Mac Mini, 24GB) focuses on memory efficiency, recommending **Gemma4:26b** for reliable tool-based loops and **Qwen2.5-Coder:14b** for high-throughput coding tasks. Tier C (RTX 3090/4090, 24GB VRAM) leverages CUDA, suggesting **Qwen3.5:35b-A3B-Coding** in NVFP4 format for efficient long-context tasks, and **Mistral NeMo:12b** for extremely fast, large-window summarization. Across all tiers, **Phi-3.5-mini:3.8b** is suggested as a lightweight model for quick experiments or low-resource scenarios.

The report concludes with critical technical recommendations for ensuring Pi-Ollama compatibility and addressing known bugs. Users are advised to use Ollama’s native `/api/chat` or set `stream: false` to prevent the agent loop from stalling due to Ollama's failure to handle `tool_calls` in streaming mode. Additionally, the report recommends explicitly setting a large `num_ctx` to prevent silent content truncation, configuring `compat.supportsDeveloperRole: false` in `models.json` to ensure system prompts are recognized, and keeping Pi extensions updated to handle the specific requirements of newer models with massive context windows.
