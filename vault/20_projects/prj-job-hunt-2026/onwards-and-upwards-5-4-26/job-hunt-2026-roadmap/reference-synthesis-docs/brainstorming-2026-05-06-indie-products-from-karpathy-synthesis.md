# Brainstorming Session — Indie Products from Karpathy Synthesis

## Topic

Generate 12+ indie product ideas that pressure-test the synthesis doc's existing 8 sell-side ideas and surface ones the doc missed — leveraging Sean's unusual asset stack against Karpathy's framework.

## Technique

**Forced Connections (Creative).** Smashing Sean's unusual assets (animation craft, crypto API access, vault knowledge loop, hooks system, hybrid router, voice-modes, granola pipeline, three-domain operating model, intent-engineering skill) against Karpathy's themes (verifiable RL envs, MCP, agent-native infra, eval design, council of judges, persistent IDs, AutoResearch, jagged intelligence) — because the doc's 8 ideas live in the obvious one-asset cells. The non-obvious products live in the collisions.

## Ideas

**Forced collisions — Sean's stack × Karpathy's themes:**

1. **`consistency-bench`** — Public benchmark for image-character-consistency. Animators submit anchor sheets, models get scored, leaderboard published. Frame-keeper sells the pipeline; consistency-bench sells the eval that labs and Adobe would license.

2. **`spec-audit`** — Spec-driven smart-contract auditor. Input: English contract spec. Output: verifiable adversarial diff vs deployed bytecode. Lives at the spec layer — catches *intent drift* traditional auditors miss.

3. **`evolving-wiki`** (vault-mcp v2) — Phase 6 loop, but synthesizer runs adversarially: each night it tries to *contradict* yesterday's articles. Only survivors persist. AutoResearch applied to personal knowledge.

4. **`replicant-test`** — Hiring product. Candidate gets a sandboxed Superuser Pack + brief + 4 hours. Hooks capture HOW they work (prompt patterns, error recovery, security choices). Sells to seed-stage AI startups who can't tell senior agentic engineers from posers.

5. **`lifeOS-bootstrap`** — Three-domain operating model (the-block / creative-studio / life-systems) as a paid setup-and-tune service. Per the doc: "your three-domain operating model is genuinely rare." Productize the rarity.

6. **`intent-to-eval`** — Doc names intent-engineering (Sean's IP) and eval design (the frontier) but never connects them. This does. Write the one-page intent spec; tool generates eval rubric, golden traces, adversarial probes the agent must pass before shipping. *The missing artifact in the entire prompt-engineering category.*

7. **`spec-lint`** — ESLint for agent specs. Catches the MenuGen bug class: email-as-ID, ambiguous boundaries, missing termination conditions, undefined error states. Open-source kernel, premium rule packs.

8. **`meshrouter`** — HybridRouter as a public MCP server. Cost + privacy + latency-aware routing across Sonnet/Haiku/Opus/Gemini/Ollama. Per the doc's "don't pin to one vendor."

9. **`mcp-shield`** — Reverse-proxy MCP server that wraps OTHER MCP servers and applies Sean's hook patterns (block-secrets, network-access-control, require-confirm-highrisk). The 9,400-server registry creates the pain; "safely use strangers' MCP servers" is the unowned enterprise category.

10. **`kept`** — Granola meeting memory agent. Continuous personal wiki of who-knows-what, who-said-what, who-promised-what, with cross-meeting contradiction detection. Beats Granola's own summaries because it spans the corpus.

11. **`creative-agent-trial`** — Hiring harness for *creative-industry* PMs. 30-second brief + agent fleet + watchable-result deadline. Sean becomes the canonical passing candidate; the test becomes inbound lead-gen for creative-PM roles.

12. **`production-diary-pack`** — Notion/Obsidian template + critique service. Every agentic-creative ships a production diary with their portfolio piece; Sean sold the format. June 11 short is the prototype.

13. **`taste-council`** — Generalize voice-modes from writing to design, animation, copy, video. MCP server any creative agent can plug into. Multi-model taste scoring with adversarial prompts. Per the doc, voice-modes is "more strategically valuable than you may have framed it" — this is what framing it looks like.

14. **`delta-brief`** — Public weekly "what's new in agentic engineering / crypto / animation" delta against vault baseline. Compounding distribution because it's a feed, not essays from scratch. Reading habit → public credential.

**100-Ideas riffs the doc doesn't see:**

15. **`karpathy-grader`** — 20-question scorecard. Founders/PMs answer; get a heatmap against Karpathy's verifiable/MCP/agent-native framework. SaaS for AI strategy clarity. Substack lead magnet.

16. **`vibe-to-spec`** — Watches the live Anti-Gravity session and emits a running spec doc + persistent-ID warnings *during* vibe-coding. Captures the spec FROM the act of coding rather than waiting for one.

17. **`undertrained-domains`** — Paid newsletter listing the week's most-promising "valuable + verifiable + undertrained" domains. Curator product. Distillation of the exact Karpathy wedge.

18. **`contradiction-feed`** — Vault's contradiction-detection (Phase D edges) as a public Substack. The Stratechery of "what I used to think vs what I think now." Differentiated content product nobody can clone because nobody else has the typed-edge schema.

19. **`agent-fleet-kit`** — Daily-driver / vault-synthesizer / knowledge-lint topology exported as a forkable repo + paid managed version. Open-source kernel, paid hosting + tuning.

20. **`career-pivot-fleet`** — Specialized fleet for industry-pivoters. Daily brief + research fleet + portfolio diary scribe + weekly plan reviewer, calibrated to "land role X by month Y." Sean's job-hunt-2026 project IS this product. Eat the dogfood, sell the recipe.

## Top Picks

Five rise. Criteria: leverages already-built IP, contrarian wedge the doc missed, 90-day shippable in some form, survives multiple market scenarios.

**1. `intent-to-eval` (#6).** The doc explicitly names two of Sean's highest-leverage IP pieces (intent-engineering + eval-as-craft) but never connects them. Sealing the loop is the most obvious, most missing artifact in the entire agentic-engineering tooling category. Sells as templates, MCP server, and a Substack series simultaneously. Demonstrates Sean's exact PM value prop. Career-portable across PM, indie, and creative-PM paths.

**2. `mcp-shield` (#9).** Doc's #1 recommendation is "ride MCP." But everyone is shipping vault-mcp / skill-servers — that's crowded. The *firewall* layer is the contrarian wedge: 9,400 strangers' servers + zero trust gateway = unowned enterprise category. Sean's hooks are the unique IP. Highest enterprise-revenue ceiling on the list.

**3. `taste-council` (#13).** Doc's adjacent observation #1 is that voice-modes is undervalued. This is what *valuing* it looks like — generalize to all creative domains, ship as MCP, create the credential the doc says Sean lacks for creative-industry PM roles. Converts 12 years of taste into an asset other people can rent.

**4. `delta-brief` (#14).** Smallest, fastest shippable, biggest compounding effect. Sean reads voraciously already; this turns the reading habit into public distribution. Beats writing 12 essays in 12 months because it's a feed, not a project. Pairs with intent-to-eval as the public surface.

**5. `spec-lint` (#7).** Of the brand-new ideas this has the cleanest "obviously useful, somehow nobody built it" energy. ESLint analogy is instantly grokkable. Open-source kernel = trust, premium rules = revenue, hook system is the prior art that makes Sean the credible builder.
