## Definition

Background processing in this context refers to the architectural pattern where AI agents perform complex computations or data synthesis asynchronously, allowing the human user to remain in a state of flow or engage in other tasks. This decouples the time of request from the time of result, enabling continuous productivity. The value lies in the invisibility of the work; the agent handles the latency and complexity, presenting only the final artifact or insight when ready.

## Context

For Sean's 'Agent Fleet' and 'Daily-driver agent' systems, background processing is essential. It allows multiple agents to work on job applications, code reviews, or knowledge synthesis simultaneously without blocking Sean's primary workflow, effectively multiplying his effective working hours.
