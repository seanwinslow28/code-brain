## What this is

Critiques from two external reasoners (gpt-5.5 via Codex CLI, Gemini 3 via Anti-Gravity CLI) of [[portfolio-walkthrough]]. The synthesizer describes what the concept is; this expansion proposes what's missing.

## From Codex (gpt-5.5)

1. **Add “Demo As Proof-Carrying Story”**
   - **Anchor:** Bret Victor, *Inventing on Principle* talk.
   - **What to add:** Portfolio walkthroughs are not just project summaries; the strongest version makes the candidate’s governing principle visible through a live artifact. Add a mode where each project is framed as: “I noticed a mismatch between what the system rewarded and what the work actually needed, so I built an environment where the right behavior became easier.”
   - **Unlocks for Sean:** A portfolio one-pager or interview talk track where Code-Brain, 16BitFit, and intent-engineering are not “cool agent projects,” but evidence of one repeatable thesis: Sean builds systems that turn vague intent into executable operating environments. This prevents the generic “I used agents to automate X” failure mode.

2. **Add “Case Method Cross-Examination”**
   - **Anchor:** Harvard Business School case method, specifically C. Roland Christensen, David A. Garvin, and Ann Sweet, *Education for Judgment: The Artistry of Discussion Leadership*.
   - **What to add:** A portfolio walkthrough should include adversarial decision points, not only outcomes. Add a pattern: “At this fork, I had three plausible moves. I rejected A because __, delayed B because __, and chose C despite __ risk.”
   - **Unlocks for Sean:** A senior-PM / AI-PM interview artifact: a “decision spine” version of each project. For Code-Brain, that means explaining why Obsidian-Git owns commits, why local LDR is bounded away from compound research, why vault critic uses multiple CLIs, and why MCP access is interactive-only. This turns the walkthrough from capability inventory into judgment evidence.

3. **Add “Ladder of Inference Walkthrough”**
   - **Anchor:** Chris Argyris, *Overcoming Organizational Defenses*; popularized in Peter Senge, *The Fifth Discipline Fieldbook*.
   - **What to add:** Portfolio walkthroughs should show how Sean moves from observation to interpretation to action without hiding the assumptions. Add a rubric for each project: observed data → selected meaning → assumption → action → validation signal → revised belief.
   - **Unlocks for Sean:** A Substack essay or interview runbook called something like “How I Debug My Own Agent Fleet.” The current concept says “explain projects”; this would let him demonstrate epistemic hygiene. It is especially useful for agentic-engineering IC roles because it shows he can inspect automation failures, avoid self-congratulatory dashboards, and convert incidents into system constraints.
