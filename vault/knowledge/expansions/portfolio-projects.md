## What this is

Critiques from two external reasoners (gpt-5.5 via Codex CLI, Gemini 3 via Anti-Gravity CLI) of [[portfolio-projects]]. The synthesizer describes what the concept is; this expansion proposes what's missing.

## From Codex (gpt-5.5)

1. **Add “Proof Artifact Taxonomy” anchored on Simon Willison’s *Things we learned about LLMs in 2024***

   Current concept says “portfolio projects” as a curated selection. That is too resume-native. Add a taxonomy that distinguishes **demo**, **trace**, **postmortem**, **benchmark**, **runbook**, and **decision record**.

   Exemplifier: Simon Willison, essay **“Things we learned about LLMs in 2024”**. The useful move is not the content itself; it is the evidence style: concrete observations, named failures, dated capabilities, reproducible examples, and lessons that survive tool churn.

   Sentence pattern to add:  
   **“This project proves X because artifact Y lets a reviewer inspect Z, not merely believe my description.”**

   Unlocks: a portfolio one-pager where Code-Brain is not “I built an agent fleet,” but a set of inspectable proof objects: nightly manifest, failed critique specimen, cost caps, launchd schedule, lint output, manual override path. This helps Sean decide which artifacts belong in job materials and which are just impressive internal machinery.

2. **Add “Workbench Narrative” anchored on Bret Victor’s *Inventing on Principle***

   The missing facet is not project selection; it is **how the project reveals the builder’s taste**. Sean’s best work is not only agentic infrastructure, it is a personal workbench for seeing, steering, and correcting systems.

   Exemplifier: Bret Victor, talk **“Inventing on Principle”**. The key reference is Victor’s principle-driven demos: he does not merely show tools; he shows a feedback loop becoming visible, manipulable, and morally necessary.

   Sentence pattern to add:  
   **“I built this because I refused to accept DELAY/OPACITY/DRIFT in WORKFLOW; the artifact makes FEEDBACK visible at the moment of decision.”**

   Unlocks: a Substack essay or portfolio case study where Code-Brain becomes an argument about working conditions for agentic builders. It would let Sean write beyond “here are my tools” into “here is the principle my tools enforce”: zero-cost overnight learning loops, critic agents that generate missing references, and intent specs that stop agents from optimizing the wrong thing.

3. **Add “Project as Boundary Object” anchored on Susan Leigh Star and James Griesemer’s *Institutional Ecology, ‘Translations’ and Boundary Objects***

   The current concept treats portfolio projects as self-expression. For PM and AI-PM roles, that is incomplete. A strong portfolio project should also be a **boundary object**: legible enough for engineers, recruiters, hiring managers, and product leaders to each extract value without needing the same vocabulary.

   Exemplifier: Susan Leigh Star and James R. Griesemer, paper **“Institutional Ecology, ‘Translations’ and Boundary Objects”**.

   Sentence pattern to add:  
   **“This artifact has four readings: recruiter signal, PM judgment, engineer credibility, and operator proof.”**

   Unlocks: a decision framework for packaging each project. For Code-Brain, the recruiter view is “AI workflow builder”; the PM view is “turns ambiguous goals into operating systems”; the engineer view is “ships Python/TypeScript/SQLite/launchd agent infrastructure”; the operator view is “runs daily with observability and failure handling.” This would produce a portfolio matrix the current concept cannot reach.
