## What this is

Critiques from two external reasoners (gpt-5.5 via Codex CLI, Gemini 3 via Anti-Gravity CLI) of [[practice-confidence-plan]]. The synthesizer describes what the concept is; this expansion proposes what's missing.

## From Codex (gpt-5.5)

1. **Add a “deliberate practice loop” anchored on K. Anders Ericsson, _Peak_**

   Current concept says “daily routine” but not how Sean will convert weak interview answers into skill gains. Add Ericsson-style deliberate practice: isolate one subskill, perform under constraint, get immediate feedback, repeat with increasing difficulty.

   Pattern to add: `One rep = prompt -> timed answer -> transcript -> scored rubric -> rewritten answer -> second timed answer`.

   This unlocks an **interview practice runbook** and an **agent spec** for a “mock-interview sparring agent” that does more than quiz vocabulary. It would let Sean train specific failure modes: rambling, overexplaining architecture, underselling PM judgment, or sounding like an agent-tools hobbyist instead of a senior operator.

2. **Add a “retrieval practice over rereading” constraint anchored on Henry Roediger & Jeffrey Karpicke, “Test-Enhanced Learning”**

   The plan currently includes feeds, vocab rotation, and targeted consumption. That risks becoming sophisticated rereading. Add the Roediger/Karpicke principle: memory and fluency improve more from forced recall than from review.

   Pattern to add: `No source open. Answer from memory first. Then compare against canon. Then patch only the missing move.`

   This unlocks a **daily interview drill artifact**: 10 closed-book prompts mapped to Sean’s AI-PM / agentic-engineering canon. It also enables a measurable decision: which concepts are actually interview-ready versus merely familiar. Current concept cannot distinguish “I recognize this” from “I can deploy this under pressure.”

3. **Add a “performance repertoire” mode anchored on Keith Johnstone, _Impro_**

   Sean’s concept is about confidence, but confidence in interviews is partly performative range: status, timing, offer/accept, recovery after a bad answer. Add a Johnstone-inspired “status and scene control” layer: practice the same technical story in different interview stances.

   Pattern to add: `Tell the same story three ways: calm senior PM, precise IC builder, candid postmortem owner.`

   This unlocks a **voice-calibration essay**, a **mock-interview prompt pack**, and stronger **portfolio narrative decisions**: when Sean should lead with Code-Brain as infrastructure, as taste, as product judgment, or as proof of autonomous execution. Without this, the plan trains content recall but not the live-room problem where he can sound overbuilt, underfocused, or too inward-facing.
