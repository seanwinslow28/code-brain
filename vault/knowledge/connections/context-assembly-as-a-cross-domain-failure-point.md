## Synthesis

The lack of comprehensive context assembly connects concepts from different domains: agent health, synthesis workflows, and daily automation routines — all of which hinge on accurate knowledge retrieval.

## Threads

### [[Agent Health Monitoring]]

> The agents I’m watching fail in production aren’t failing because the retrieval method is wrong — they’re failing because the retrieval system can’t assemble what the agent actually needs before it starts acting.

### [[Indexing and Synthesis]]

> The relevant retrieval object isn’t 'a chunk.' It’s a structured bundle of context.

### [[Automation Routines]]

> The agent burns tokens and wall-clock time rebuilding context every run, and when the answer finally lands, it lands confidently — which is the most expensive way to be wrong.

## Implications

- Improving context assembly could reduce automate failure rates in all domains — from knowledge systems to daily agent routines.
- This points toward a need for better cross-domain synthesis tools that feed into all agent types.
