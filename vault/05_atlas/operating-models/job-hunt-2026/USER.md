# USER — Job Hunt 2026

## Decisions Made 3+ Times Weekly

**Strategy / building decisions (the dominant set):**
- "What do I have to build today?"
- "What are hiring managers / companies actually looking for in an AI/Tech PM right now?"
- "How can I stand out from the crowd?"
- "What projects would impress most?"
- "What's the move that puts me multiple steps ahead?"

**Pipeline / triage decisions:**
- Apply to this role or skip?
- Reply to this recruiter now, or batch for the Friday retro?
- Spend the next 2 hours on Track-C MCP / resume polish / applications / outreach?
- Reach out to this warm-network person now, or wait until I have a sharper ask?
- Take this recruiter screen call or pass?

## Real Prioritization Criteria

_(Sean's actual filters, not the polished-resume version.)_

1. **Geo fit:**
   - **Auto-yes:** fully remote.
   - **Yes:** Boston metro hybrid (1–3 days/week in office).
   - **Auto-no:** 5-days-in-office. Already done it; it's a nightmare. Non-negotiable.
2. **Warm intro available?** If yes, weight goes way up. Past wins have been referral-driven; cold apps are the lower-ROI path by lived experience.
3. **Company brand / size / stability.** Post-Block, Sean wants established + well-known + decent headcount + good ratings. Not buying any more "scrappy startup that surprise-layoffs you" energy.
4. **Archetype:** AI PM > Tech PM > Creative PM (priority order; AI is the strongest target).
5. **Salary above $100k walk-away.**
6. **Learn + grow potential.** Will I get better at AI PM craft in this role?
7. **Energy at the moment of the decision** (legitimate filter — fatigue inflates "skip" rates).

## Auto-Yes

- Warm-intro outreach (always).
- Any recruiter mentioning an AI-native company.
- Big-name / well-known companies.
- Any role with comp clearly above $100k/yr.
- Any place that explicitly enables learning + growth.

## Auto-No

- Cold-recruiter spam below the walk-away.
- Non-Boston-metro non-remote roles.
- 5-days-in-office anything.
- Bootcamps / consulting.
- Anything below $100k/yr base.

## Daily Investment Heuristic

- **Default:** resume yesterday's open thread first. Each evening Claude writes the next-day kickoff prompt; that prompt is the morning's first move.
- **Project pipeline feeder:** deep_researcher + Obsidian web clipper run between sessions to surface new portfolio-worthy projects, AI trends, and "what's getting recognition this week" signals.
- **No vibe-driven start.** The kickoff prompt and the open thread together mean "what do I work on first" is decided the night before, not at 9:30 AM.

## Agent Delegation

**Already trusted to agents:**
- Scrape + discover (job postings, AI-trend signals, rising-company watchlist).
- Plan execution + sequence work.
- Be a thinking partner — challenge, sharpen, propose.
- Build alongside Sean — write code, draft docs, scaffold MCP servers.
- Teach enterprise-grade patterns Sean hasn't yet built at scale.
- Generate tomorrow's kickoff prompt at end-of-day.
- Triage labeled email into vault markdowns (once Gmail-label pipeline is built).

**Never delegated:**
- Outbound messages to humans (recruiters, warm network, references, hiring managers).
- Application submissions.
- Any direct conversation with a recruiter.
- LinkedIn / Substack posts that go out under Sean's name without Sean's review-and-publish.
- Final "yes/no" on a role or offer.

**Boundary rule:** Every word that hits another human's inbox is Sean's. Agents draft; Sean sends.

## Tiebreakers When Unsure

1. Warm intro wins.
2. Bigger brand wins.
3. **Weekly research feed** — surface 2–3 rising AI companies that aren't household names yet but are accelerating. If a tie comes down to "boring big-name vs rising AI co," the rising AI co can win, but only when it's been validated by the weekly research pass.
4. If still tied, go with the one Claude can help ship faster (lower friction wins).

## Definition of "Done" in This Domain

| Unit of work | Done means |
|---|---|
| Application | submitted **and** logged in the pipeline table |
| Outreach | sent (no reply or call required to close the loop) |
| Resume variant | self-approved + ready-to-attach |
| LinkedIn post | written, reviewed, ready to publish |
| Substack article | written, reviewed, ready to publish (covering an AI topic or a build-in-public update) |
| MCP server v0 | pushed public **and** polished **and** portfolio-ready (README + demo + showcase frame) |
| Track-C build session | committed + pushed + journaled in the daily note |
| Friday retro | numbers filled, what-worked / what-didn't filled, headline goal for next week set |
