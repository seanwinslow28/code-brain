## Grader bugs found and fixed

None — every failure is REAL.

**Anomaly note (not a grader bug):** The `pytest_sessionfinish` and `pytest_runtest_logreport` hooks defined at the bottom of `runner.py` do not fire when pytest runs the file as a test module (only conftest.py hooks are auto-registered by pytest's plugin system). As a result, `last-run.md` remains empty after each run. The grading logic itself (pass/fail/skip verdicts) is correct and reproducible; only the autogenerated summary table is broken. This is a known limitation that should be fixed by moving the hooks to `conftest.py` or registering `runner.py` as a plugin, but it does not affect the baseline classification. The `last-run.md` written here is manually derived from the captured log.

## Anomalies

- **vs-014 vs vs-015 coupling:** Both cases run against the same fixture corpus (6-note fixtures, no LLM mock). They catch two distinct failure modes of the same root bug: vs-015 catches the wrong status label (`ok` when output is empty); vs-014 catches that no output was produced at all. vs-014 will remain FAIL after B2 (status fix) and only flips green in B6 when a real LLM caller produces articles. This is expected and documented.

- **vs-016 and vs-017 share a root cause:** Both require a `llm_caller_mock` that raises `ConnectionRefusedError`. The runner's `_invoke_synthesizer` doesn't wire up the mock yet (Workstream B adds this plumbing). Both cases currently fail because the synthesizer falls through to `status='ok'` rather than escalating. The mock-wiring infrastructure (B2/B3) will surface the intended failures cleanly.
