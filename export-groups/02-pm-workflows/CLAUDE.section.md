## PM Workflows

Product management acceleration skills. All PM skills use an interview-first approach: Claude asks clarifying questions before generating output.

### Available Skills
- **quick-prd**: Rapid PRD generation ("create a PRD for user auth")
- **ticket-batch**: Batch Jira/Linear ticket creation from specs ("create tickets for this PRD")
- **stakeholder-brief**: TL;DR-first status updates ("write a stakeholder update")
- **tech-spec**: Implementation-ready technical specs ("write a tech spec for the API")
- **meeting-prep**: Structured agendas and prep docs ("prep me for the sprint review")
- **decision-doc**: Document decisions with rationale ("document the auth approach decision")

### Available Agents
- **doc-reviewer**: Reviews PRDs and specs for completeness (read-only)
- **data-analyst**: Data analysis and visualization (read-only by default)

### Templates
Templates are folded into skill `references/` directories. Claude reads them automatically when generating output.
