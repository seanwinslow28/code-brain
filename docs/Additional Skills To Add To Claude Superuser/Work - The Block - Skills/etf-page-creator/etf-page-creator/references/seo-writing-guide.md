# The Block's ETF SEO Writing Guide

This reference provides complete patterns and examples for writing SEO titles and meta descriptions for The Block's ETF pages.

## SEO Title Format

### Standard Pattern
```
[ETF Name] ([Symbol]) [Status Keyword] & Key Details | The Block
```

### Status Keywords
- **"Live Status and Key Details"** - Most common, use for active trading ETFs
- **"Status & Key Details"** - Shorter alternative, equally acceptable

### Rules
1. Always include ticker symbol in parentheses after ETF name
2. Always end with "| The Block" (pipe separator + site name)
3. Use ampersand (&) not "and" before "Key Details"
4. Capitalization: Title Case for ETF name, uppercase for symbol

### Complete Examples

**Spot ETFs:**
- Grayscale Cardano Trust (GADA) Live Status and Key Details | The Block
- Fidelity Solana Fund (FSOL) Status & Key Details | The Block
- Canary HBAR (HBR) Status & Key Details | The Block

**Futures ETFs:**
- Teucrium XRP ETF (XXRP) Live Status and Key Details | The Block
- ARK 21 Shares Ethereum Futures Strategy ETF (ARKZ) Live Status and Key Details | The Block

**Leveraged ETFs:**
- 21Shares Dogecoin ETF (TXXD) Status & Key Details | The Block

### Character Count Considerations
- Target: 50-60 characters (Google displays ~60 chars)
- Maximum: 70 characters before truncation
- The "| The Block" suffix adds 13 characters
- Adjust ETF name length if needed to stay under 70 total

## Meta Description Format

### Standard Patterns

#### Pattern 1: Standard Format (Most Common)
```
[ETF Name] ([Symbol]) is a [type] [asset] ETF [launched/proposed] by [Issuer]. See [status phrase], [statistics,] and details.
```

#### Pattern 2: Overview Format (Alternative)
```
An overview of the [Full ETF Name] ([Symbol]) with live chart, latest price, market info and related ETF data.
```

### Type Descriptors

**By Structure:**
- "spot [asset]" - For physical-backed ETFs
- "futures [asset]" - For futures-based ETFs
- "2x Long Daily [asset]" - For leveraged ETFs
- "[asset]" - Generic when structure not specified

**By Status:**
- "launched by" - For approved/live ETFs
- "proposed by" - For pending ETFs
- "potential" - For very early stage proposals

### Status Phrase Options
- "See ETF live status, statistics, and details" (most common for live ETFs)
- "See live status and details" (for pending/new ETFs)
- "See the latest status, details and statistics here" (alternative)

### Complete Examples by Category

**Pending Spot ETF:**
```
The Grayscale Cardano Trust is a potential spot Cardano (ADA) ETF proposed by Grayscale. See live status and details.
```
- Uses: "potential", "proposed by"
- No "statistics" (not live yet)
- ~135 characters

**Live Spot ETF:**
```
Fidelity Solana Fund (FSOL) is a spot SOL ETF launched by Fidelity. See ETF live status, statistics, and details.
```
- Uses: "launched by"
- Includes "statistics"
- ~130 characters

**Live Leveraged ETF:**
```
The 21Shares 2x Long Dogecoin ETF (TXXD) is a Dogecoin ETF launched by 21Shares. See the latest status, details and statistics here.
```
- Uses: "launched by", "2x Long" descriptor
- Alternative ending phrase
- ~150 characters

**Futures ETF:**
```
The Teucrium 2x Long Daily XRP ETF is a futures XRP ETF launched by Teucrium. See ETF live status and details.
```
- Uses: "2x Long Daily" (full leverage descriptor)
- "futures XRP ETF" (type + asset)
- ~125 characters

**Overview Style (Alternative):**
```
An overview of the ARK 21 Shares Active Ethereum Futures Strategy ETF (ARKZ) with live chart, latest price, market info and related ETF data.
```
- Uses different structure entirely
- More descriptive, includes features
- ~160 characters

## Writing Decision Tree

### For SEO Title:
1. Start with full ETF name
2. Add symbol in parentheses
3. Choose status keyword ("Live Status and" or just "Status")
4. End with "& Key Details | The Block"
5. Check character count (<70 chars)

### For Meta Description:

**Step 1: Determine ETF Status**
- Pending/Proposed → Use "potential" + "proposed by"
- Approved/Live → Use "launched by"

**Step 2: Identify ETF Type**
- Direct crypto ownership → "spot [asset]"
- Futures contracts → "futures [asset]"
- Leveraged exposure → "2x Long [Daily] [asset]"

**Step 3: Choose Ending**
- Live ETF → "See ETF live status, statistics, and details"
- Pending ETF → "See live status and details"
- Alternative → "See the latest status, details and statistics here"

**Step 4: Construct**
```
[Name] ([Symbol]) is a [type] [asset] ETF [launched/proposed] by [Issuer]. [Ending].
```

**Step 5: Validate**
- Character count: 150-160 (optimal for Google)
- Includes symbol in parentheses
- Includes issuer name
- Clear call-to-action ("See...")

## Asset Name Abbreviations

Use ticker abbreviations in meta descriptions for brevity:
- Bitcoin → BTC (in some contexts, spell out in others)
- Ethereum → ETH
- Cardano → ADA
- Solana → SOL
- XRP → XRP
- Dogecoin → Spell out or DOGE
- Hedera → HBAR

**Rule:** First mention in meta description should match what users search (often full name), symbol in parentheses

## Common Variations by Issuer

### Grayscale
- Typically uses "Trust" not "ETF" in name
- Example: "Grayscale Cardano Trust (GADA)"
- Structure: "[Grayscale] [Asset] Trust"

### Fidelity
- Uses "Fund" in name
- Example: "Fidelity Solana Fund (FSOL)"
- Structure: "Fidelity [Asset] Fund"

### ARK/21Shares
- Often have longer, descriptive names
- Example: "ARK 21 Shares Active Ethereum Futures Strategy ETF"
- May include "Active", "Strategy", partner names

### Teucrium
- Known for leveraged products
- Example: "Teucrium 2x Long Daily XRP ETF"
- Always includes leverage ratio in name

### 21Shares (Solo)
- Separate from ARK partnership
- Example: "21Shares 2x Long Dogecoin ETF (TXXD)"
- Often leveraged products

Before finalizing SEO content, verify:

**SEO Title:**

- [ ] Symbol in parentheses immediately after name
- [ ] Status keyword present ("Live Status" or "Status")
- [ ] Ends with "& Key Details | The Block"

- [ ] Uses title case appropriately

**Meta Description:**
- [ ] 150-160 characters
- [ ] Includes symbol in parentheses
- [ ] Specifies ETF type (spot/futures/leveraged)
- [ ] Names the underlying asset
- [ ] Uses correct status verb (launched/proposed)
- [ ] Includes issuer name
- [ ] Ends with clear call-to-action
- [ ] No grammatical errors
- [ ] Matches The Block's style

## Edge Cases & Special Situations

### Multi-Asset ETFs
If ETF tracks multiple cryptocurrencies:
```
[Issuer] Crypto Basket ETF ([SYMBOL]) is a multi-asset crypto ETF launched by [Issuer]. See ETF live status, statistics, and details.
```

### International ETFs (Non-US)
Maintain same format but may specify region:
```
[Name] ([Symbol]) is a [region] spot [asset] ETF launched by [Issuer]. See ETF status and details.
```

### Inverse/Short ETFs
Include descriptor:
```
[Name] ([Symbol]) is an inverse [asset] ETF launched by [Issuer]. See ETF live status and details.
```

### Index-Tracking ETFs
Mention the index if relevant:
```
[Name] ([Symbol]) tracks the [Index Name] and is launched by [Issuer]. See ETF live status and statistics.
```

## Optimization Tips

**For Search Rankings:**
1. Include cryptocurrency name in both title and description
2. Use ticker symbol consistently (how users search)
3. Include "ETF" keyword in meta description
4. Natural language (not keyword stuffing)

**For Click-Through Rate:**
1. Clear value proposition ("See live status, statistics")
2. Issuer credibility (name well-known firms)
3. Specific details (type, asset)
4. Active voice

**For User Experience:**
1. Match user search intent
2. Accurate status information
3. No misleading claims
4. Consistent formatting across all ETF pages
